# 2026-06-10 — Kickoff: unificar rev-dev-frontend + revdev2.0

## Contexto

Existían dos proyectos del mismo producto **RevDev**:

- `rev-dev-frontend` — Vite + React Router + styled-components. Copy más rico
  (servicios, proceso 4 pasos, works, Ideas, Case).
- `revdev2.0` — Next.js 13 + Tailwind + framer-motion. Esqueleto moderno,
  Header/Footer/Hamburg, páginas Us y Careers.

Decisión: fusionar ambos en **un solo proyecto Next.js** con librerías al día.

## Decisiones tomadas hoy

- **Stack**: Next.js 15 (App Router) + TS + Tailwind v4 + framer-motion +
  next-intl + Prettier. Se descartan Vite, React Router y styled-components.
- **Idioma**: multi-idioma (i18n) con `en`, `es`, `de`. Default `en`.
- **Objetivo**: agencia (clientes) + studio (proyectos/ideas) + reclutar.
- **Páginas**: Home, Services, Us, Careers, Ideas, y dos nuevas — **Projects**
  (disponibles + en trabajo) y **Partners** (socios + propuestas).
- **Estilo**: Tailwind + framer-motion.
- **Ubicación**: proyecto unificado en `/Users/rev/dev/rev-dev/rev-dev/`.
- **Docs**: `docs/BLUEPRINT.md` (documento vivo) + `docs/history/` con archivos
  fechados `YYYY-MM-DD-tema.md` como línea de historia.

## Siguiente paso

Scaffold del proyecto Next.js 15 con la estructura definida en el BLUEPRINT.
