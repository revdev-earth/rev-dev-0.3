# 2026-06-10 — Scaffold del proyecto unificado

## Hecho

Creado el proyecto Next.js en `/Users/rev/dev/rev-dev/rev-dev/`.

- **Stack instalado**: Next.js 16.2.9, React 19, Tailwind v4, TypeScript 5,
  `motion` 12, `next-intl` 4, `resend` 6, Prettier + plugin tailwind.
- **i18n**: `next-intl` con segmento `[locale]` (en/es/de), default `en`,
  `localePrefix: "always"`. Archivos `i18n/{routing,navigation,request}.ts`,
  `middleware.ts`, plugin en `next.config.ts`.
- **Mensajes**: `messages/{en,es,de}.json` con todo el copy (Nav, Home, Services,
  Process, Projects, Ideas, Partners, Us, Careers, Footer).
- **Layout**: `app/[locale]/layout.tsx` (html/body + Header/Footer + provider).
  No hay root layout — el de `[locale]` es la raíz (patrón next-intl).
- **Páginas**: Home (Hero animado + servicios + proceso + CTA), Services
  (+ formulario contacto), Projects (disponibles / en trabajo), Ideas (stub),
  Partners (socios + propuestas), Us (8 bloques), Careers (vacantes + perks).
- **Componentes**: Header (nav + hamburguesa + LocaleSwitcher), Footer, Hero,
  ContactForm, LocaleSwitcher.
- **Data estática**: `lib/projects.ts`, `lib/partners.ts`.
- **Contacto**: `app/api/contact/route.ts` con Resend. Sin `RESEND_API_KEY`
  loguea y responde ok (modo dev). `.env.example` documenta las vars.

## Decisión sobre idioma en URL

Se evaluó manejarlo interno (cookie). Se eligió **prefijo siempre** por SEO
multi-idioma, links compartibles por idioma y simplicidad de cache/SSR.
